Multi-agent public-opinion intelligence for evidence-grounded analysis and traceable reports

CapstoneProject is an end-to-end system for public-opinion analysis. It gathers web, media, and social evidence; compares stance and trust signals; detects divergence, blind spots, and echo-chamber signals; then turns the analysis into editable, exportable reports through Signal Studio.

📁 View source code on GitHub 🏛️ HKU · Supervisor: Prof. Kai Han

📋 Project Brief

Public Opinion Analysis System for Social Media is a multi-agent intelligence system that converts an analysis brief into a traceable, evidence-grounded public-opinion report.

Given a user-defined topic, the automated pipeline:

  • Gathers public evidence from multiple search providers and social platforms
  • Scores and deduplicates collected sources using trust and relevance metrics
  • Classifies stance, sentiment, and platform context for each piece of evidence
  • Compares source groups and quantifies divergence across stances and platforms
  • Conducts structured multi-perspective deliberation to separate consensus from dissent, identify coverage gaps, and flag underlying tensions
  • Persists a structured coordinator artifact that captures the full reasoning trace
  • Generates an editable report with deterministic rendering to HTML, Markdown, and PDF

⚙️ System Architecture & Technical Details

The system is organized into five layers, from interface to quality assurance.

🖥️ Interface — Signal Studio views, report editor, evidence review, runtime controls (frontend/, templates/index.html).
🔌 API Gateway — Flask routes, background tasks, config editing, static shell, observability (app.py).
🧠 Coordination — Parallel agent execution, divergence, deliberation, synthesis, artifact export (AgentCoordinator/).
📡 Evidence Engines — Query planning, search, stance, trust, social enrichment, media research (QueryEngine/, MediaEngine/, MindSpider/).
📄 Reporting — Template selection, chapter generation, Document IR, HTML/MD/PDF export (ReportEngine/).
✅ Quality — Regression tests, provider evaluation, sanitization, schema validation (tests/, api_evaluation/).
📘 AgentCoordinator — Deliberation Engine

Three-phase structured reasoning:

  • Phase 1 – Independent analysis — Each perspective analyses evidence independently, producing arguments, confidence scores, and data gaps.
  • Phase 2 – Cross-examination — Perspectives agree, challenge, or supplement one another, leading to revised positions, emerging consensus, and persistent disagreements.
  • Phase 3 – Synthesis arbitration — Consolidates consensus findings, persistent disagreements, confidence assessment, and unresolved unknowns.

Cross-Source Divergence Matrix (CSSD) — Quantifies stance distribution differences between sources using cosine similarity on stance vectors.

CRAG‑style gap filling — When important evidence is missing and search rounds remain, the system routes to a targeted search node for supplementary retrieval.

Echo chamber & bias detection — Flags evidence concentrated in a single stance or platform, generating echo warnings and silent‑majority hypotheses.

Fact‑opinion separation — Splits content into three distinct layers: verified facts with identifiable sources, subjective opinions and platform sentiments, and analytical frameworks spanning technical, economic, historical, sociological, or political lenses.

📗 ReportEngine — Document IR & Export Pipeline

Deterministic academic‑style report generator — no LLM hallucination, ensuring reproducible outputs in Markdown, HTML, and PDF.

  • Document IR — A structured intermediate representation with chapters, sections, evidence citations, and annotations.
  • Renderers — Separate engines for HTML (editable in the review board), Markdown, and PDF.
  • SSE streaming — Real‑time report generation progress via the /api/report/stream/{task_id} endpoint.

Report templates are dynamically loaded from ReportEngine/report_template/ at runtime.

🖥️ Signal Studio — Final Operator UI

Integrated workspace served by Flask via templates/index.html and static/signal-studio/.

  • Home — Topic launch, runtime status, high‑level confidence, source count, and tension indicators.
  • Readout — Executive synthesis, top insights, tensions, and follow‑up investigation cues.
  • Proof — Evidence table, stance mix, source trust, divergence heatmap, and platform readings.
  • Edit — Report generation, editable report content, annotations, and export controls.
  • Monitor — Local trace replay, runtime diagnostics, artifact metadata, feedback history, and LangSmith state.
  • Settings — Provider keys, model names, search configuration, tracing settings, and runtime start.
📊 Coordinator Output Schema (v1) — Standardized JSON fields: divergence_matrix, deliberation, gap_filling, platform_interpretations, bias_analysis, fact_opinion_separation, synthesis, source_data, coordinator_trace → ensures reproducibility and structured consumption by the ReportEngine.

Quick Start

Use Python 3.11 or newer. If python is not on PATH, use Conda or the uv commands shown in the full Setup Guide.

python -m venv .venv .\.venv\Scripts\Activate.ps1 python -m pip install --upgrade pip pip install -r requirements.txt python -m playwright install chromium Copy-Item .env.example .env

Configure provider keys in .env, then build the final UI:

cd frontend npm install npm run build cd ..

Start the Flask runtime:

python app.py

uv equivalent:

uv run --with-requirements requirements.txt python app.py

Open:

http://127.0.0.1:5000

For the complete setup path, provider profile, artifact review path, and verification checklist, use the Setup Guide.
