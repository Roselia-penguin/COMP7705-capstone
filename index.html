⚙️ System architecture & technical details

Our platform integrates four specialised agents orchestrated by a debate‑driven coordinator:

🔎 Query Agent – Stance‑aware retrieval with 4‑stance decomposition (official/support/oppose/neutral), TrustScore credibility scoring (domain authority 0.30 + timeliness 0.25 + content quality 0.25 + RRF relevance 0.20), MinHash deduplication, and MindSpider social‑media enrichment (Weibo, Zhihu, Bilibili).
🎥 Multimodal Agent – LangGraph pipeline: report_structure → process_paragraph loop → finalize_report. Supports Bocha (multimodal search) and Anspire (Chinese web/content) backends; outputs Markdown‑format media reports.
🗣️ Agent Coordinator – LangGraph deliberation engine: parallel agent execution → data bridging (BridgedProposition) → CSSD divergence matrix → multi‑perspective deliberation → gap filling → bias detection → synthesis.
📊 Report Agent – Deterministic academic‑style report generator (no LLM hallucination). outputs: Markdown, HTML, PDF. Structured fields include divergence_matrix, deliberation phases, gap_filling, fact_opinion_separation, and bias_analysis.
📘 Milestone 1: Query Agent & Multimodal Agent (Apr–May 2026)

Query Agent core innovations:

  • TrustScore – Weighted credibility scoring (domain authority 0.30, timeliness 0.25, content quality 0.25, RRF relevance 0.20).
  • Adaptive stance coverage – Stance Coverage Score (SCS) triggers gap‑filler for insufficient stance diversity.
  • MindSpider integration – Local social‑media evidence layer with platform probing (Weibo/Zhihu/Bilibili), post/comment classification, and temporal sentiment tracking.

  • Validation test – DeepSeek model release topic: 65 social posts, 10 comments, CSSD = 0.258 between web/news and social platforms.

Multimodal Agent (LangGraph) – Graph topology: START → report_structure → process_paragraph loop → finalize_report → END. Supports dual search backends (Bocha multimodal + Anspire AI search). Text processing utils extracted Structured Markdown output

📗 Milestone 2: AgentCoordinator & Report Agent (May 2026)

Coordinator deliberation engine – three‑phase structured reasoning:

  • Phase 1 – Independent analysis – Each perspective analyses evidence independently → perspective‑specific arguments, evidence, confidence, data gaps.
  • Phase 2 – Cross‑examination – Perspectives agree/challenge/supplement → revised positions, emerging consensus, persistent disagreement.
  • Phase 3 – Synthesis arbitration – Consensus findings, persistent disagreements, confidence assessment, unresolved unknowns.

Cross‑Source Divergence Matrix (CSSD) – Quantifies stance distribution differences between sources. CSSD(A,B) = 1 - cosine_similarity(stance_vector_A, stance_vector_B).
0.0–0.1: nearly identical 0.1–0.3: low divergence 0.3–0.6: moderate 0.6–0.8: high 0.8–1.0: extreme
Example (DeepSeek test): Zhihu vs Media Agent CSSD = 0.7226 (high divergence); Bilibili vs Tieba CSSD = 0.0010 (nearly identical).

CRAG‑style gap filling – If important evidence missing and search rounds remain, routes to targeted_search_node for supplementary retrieval → corrective retrieval loop.

Echo chamber & bias detection – Checks if evidence is overly concentrated in one stance/platform → produces echo warnings and silent‑majority hypotheses in final report.

Fact‑opinion separation – Three layers: Verified facts (identifiable sources), Opinions/sentiments (platform attitudes), Analytical frameworks (technical/economic/historical/sociological/political interpretation).

Academic Report Generator – Deterministic converter (no additional LLM hallucination). Report sections: Abstract, Introduction, Methodology, Data Overview, Findings, Multi‑Perspective Deliberation, Bias Assessment, Conclusions, Appendices (source list, CSSD values, coordinator trace).

Validation: 45 QueryAgent sources, coverage=1.00, 6 divergence sources → 15 matrix pairs, 4 deliberation perspectives, 7+ consensus points, 6+ dissent points, 4+ verified facts.

Architecture: Flask (port 5000) + vanilla HTML/CSS/JS + Socket.IO + Streamlit subprocesses.

  • Unified control panel – Single‑page dashboard hosting all subsystems.
  • LLM Configuration Panel – Bidirectional .env sync; API keys/base URLs/model names configurable without restarting.
  • Embedded Streamlit apps – Media Agent (:8502), Query Agent (:8503) embedded via iframe; tab switching.
  • ForumEngine integration – Background multi‑agent discussion forum; chat bubbles rendered from log parsing.
  • ReportEngine bridge – Generate + download (HTML/PDF/MD); preview rendered in iframe.
  • Real‑time console logging – Socket.IO streaming with per‑app log buffers (max 200 lines/app).
